Challenge

In this challenge we get a program that has the option mix which receives data and mix it with the flag

from hashlib import sha256
import os
from utils import listener


FLAG = b"crypto{???????????????????????????????}"


def _xor(a, b):
    return bytes([_a ^ _b for _a, _b in zip(a, b)])

def _and(a, b):
    return bytes([_a & _b for _a, _b in zip(a, b)])

def shuffle(mixed_and, mixed_xor):
    return bytes([mixed_xor[i%len(mixed_xor)] for i in mixed_and])


class Challenge():
    def __init__(self):
        self.before_input = "Oh no, how are you going to unmix this?\n"

    def challenge(self, msg):
        if "option" not in msg:
            return {"error": "You must send an option to this server."}

        elif msg["option"] == "mix":
            if not "data" in msg:
                return {"error": "Please send hex-encoded data"}

            data = bytes.fromhex(msg["data"])
            if len(data) < len(FLAG):
                data += os.urandom(len(FLAG) - len(data))

            mixed_and = _and(FLAG, data)
            mixed_xor = _xor(_xor(FLAG, data), os.urandom(len(FLAG)))

            very_mixed = shuffle(mixed_and, mixed_xor)
            super_mixed = sha256(very_mixed).hexdigest()

            return {"mixed": super_mixed}

        else:
            return {"error": "Invalid option"}


"""
When you connect, the 'challenge' function will be called on your JSON
input.
"""
listener.start_server(port=13402)

Solution

To solve this challenge we first need to find what we have control over in this “mixing” algorithm. We can see that we have control over the mixed_and, if we send data = 0 then mixed_and = 0. Then in the shuffle function all the values will be equals to the mixed_xor[0] which we do not have control over (being xored with random bytes).

First we will find a way to get the flag length from the program. We will iterate until 100 (a big number I choose that the flag size will be lower than it), and in each iteration we will create $i$ zero bytes.

If the current $i$ is lower than the flag size, the program will add random bytes to the zero bytes, resulting in a non zero data. Otherwise, the whole data will be zero, so the whole mixed_and = 0 so shuffle = mixed_xor[0], so we will receive

SHA256(mixed_xor[0] * i)

We have 256 possible values for mixed_xor[0] so we will bruteforce it to check wether the hashes are equals or not!

Because there is a random element in the remote program, it might produce mixed_xor that some bytes in it are equals to other, so we might get false positive, we will filter that using statistics (running it 5 times :) )

found = False

for i in range(100):
    zeros_block = b"\x00"*i

    # Validate 3 times because of random
    success_count = 0
    number_of_trials = 5
    for k in range(number_of_trials):

        json_send({"option": "mix", "data": zeros_block.hex()})
        mixed = json_recv()["mixed"]
        
        for j in range(256):
            data_vector = j.to_bytes(1, "little") * i

            if hashlib.sha256(data_vector).hexdigest() == mixed:
                success_count += 1

    if success_count >= ((number_of_trials * 3) // 5):
        found = True
            
    if found:
        break
    
flag_length = i

So now we got the flag size which is 39. Using the same method we can bruteforce each bit in the flag.

We will create data of 39 bytes and we will set there one of the bits, because the mixed_and is an and operation, if this specific flag bit is 1 then the mixed_and will not be zero, making the very_mixed to not equals to xor_mixed[0] * size.

What if the flag bit is 0? then the mixed_and will be zero, making the very_mixed to be equals to xor_mixed[0] * size.

Because we don’t know the value of xor_mixed[0] but we have only 256 possible values for it, we can bruteforce it!
