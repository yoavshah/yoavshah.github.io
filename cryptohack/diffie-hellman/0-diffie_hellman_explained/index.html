Here I will explain the Diffie-Hellman algorithm and the discrete log problem!

Diffie-Hellman

In the algorithm first Alice generates ahead of time a prime number p and a generator g.

Then when Bob starts a converstation with Alice, Alice generates a random value a, calculates A = pow(g, a, p), and then sends to Bob (g, p, A).

Bob generates a random value b, calculates B = pow(g, b, p), and then sends it to Alice.

So the only parameters being sent through the public unsecured channel are g, p, A, B.

To calculate the shared_secret, Alice calculates shared_secret = pow(B, a, p) and Bob calculates shared_secret = pow(A, b, p).

The shared_secret will be the same because

A = pow(g, a, p)
shared_secret = pow(A, b, p) = pow(pow(g, a, p), b, p)

B = pow(g, b, p)
shared_secret = pow(B, a, p) = pow(pow(g, b, p), a, p)

$$ (g^a)^b = (g^b)^a = g^{ab} $$

The strength of this algorithm comes from that using the parameters that are exposed to the public channel (g, p, A, B) an attacker cannot forge shared_secret!

Discrete-Log

The Discrete-Log problem is that when given a, b, p calculate x such that

$$ a^x \equiv_p b $$

Which is $$ log_a(b) \bmod p $$

In normal $log$, to calculate the $log_a(b)$ by trial and error, for example to calculate $log_2(1000)$, we can try $2^{10} = 1024$ and $2^9 = 512$ so we know the solution is between 9 to 10, but is much much closer to 10.

In fields it is not straight forward, it is much more difficult.

Diffie-Hellman and the Discrete-Log

An eavesdropper can gather the publich channel DH parameters (g, p, a, B) and if has an efficient solution to the discrete-log problem, he can calculate $$ a \equiv_p log_g(A) $$

After he calculates a, he can just calculate the shared_secret!

So we can conclude that solving the Discrete-Log problem will cause Diffie-Hellman lose its security powers.

Solving the Discrete-Log

There are many algorithms for solving the discrete log problem.

  • Baby step Giant step (BSGS)
  • Pohlig-Hellman
  • etc…

Yet they are not enough efficient to crack Diffie-Hellman with an enough key length (bit size)

There are also misuses of Diffie-Hellman when choosing bad parameters which can lower the strength of the algorithm.
