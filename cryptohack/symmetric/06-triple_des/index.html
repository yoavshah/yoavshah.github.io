Challenge

In this challenge we get a program that can encrypt any plaintext with a chosen key, and also can encrypt the flag with a chosen key. We can’t reproduce the decryption routine in our computer, because we don’t have access to the generated IV.

Remember we also can’t just take the encrypted flag and put it in the encrypt method because the DES encryption block and the DES decryption block are not the same (Or are they ? :) )

from Crypto.Cipher import DES3
from Crypto.Util.Padding import pad


IV = os.urandom(8)
FLAG = ?


def xor(a, b):
    # xor 2 bytestrings, repeating the 2nd one if necessary
    return bytes(x ^ y for x,y in zip(a, b * (1 + len(a) // len(b))))



@chal.route('/triple_des/encrypt/<key>/<plaintext>/')
def encrypt(key, plaintext):
    try:
        key = bytes.fromhex(key)
        plaintext = bytes.fromhex(plaintext)
        plaintext = xor(plaintext, IV)

        cipher = DES3.new(key, DES3.MODE_ECB)
        ciphertext = cipher.encrypt(plaintext)
        ciphertext = xor(ciphertext, IV)

        return {"ciphertext": ciphertext.hex()}

    except ValueError as e:
        return {"error": str(e)}


@chal.route('/triple_des/encrypt_flag/<key>/')
def encrypt_flag(key):
    return encrypt(key, pad(FLAG.encode(), 8).hex())

Solution

DES is a symmetric block encryption algorithm that uses 8 bytes for its blocks and keys.

Triple DES as the name suggest is 3 times the DES algorithm therefore it uses 24 bytes key (3 DES keys) and it encrypt data as follow

$$ ciphertext = E_{k3}(D_{k2}(E_{k1}(plaintext))) $$

and the decryption routine is $$ plaintext = D_{k1}(E_{k2}(D_{k3}(ciphertext))) $$

In DES there is a problem of weak keys, weak keys make the cipher behave in some undesirable way. The weak keys in DES make the encryption routine work as the DES decryption routine.

This mean that whenever a weak key is used in DES: $$ plaintext = E_{weak-key}(E_{weak-key}(plaintext)) $$

You can read more about weak keys in DES here

You might think that we can set the 3 keys of Triple DES to the same weak key, that way we can surely get the flag, but the python library protects us from using the same key in Triple DES

{"error":"Triple DES key degenerates to single DES"}

The reason is that if the same key is used 3 times in Triple DES the whole algorithm becomes just DES.

$$ ciphertext = E_{k}(D_{k}(E_{k}(plaintext))) = E_{k}(plaintext) $$

Therefore we need at least 2 different keys. We can use 2 weak keys for DES to build a weak key for Triple DES.

Let’s create the keys

$K1 = K3 = 0000000000000000$ $K2 = FEFEFEFEFEFEFEFE$

Now let’s use the encrypt_flag with this key, we should receive this output.

$$ E_{k3}(D_{k2}(E_{k1}(flag))) $$

Now let’s use the encrypted flag we got and send it to the encryption routine with the same key. We should get something like that

$$ E_{k3}(D_{k2}(E_{k1}(E_{k3}(D_{k2}(E_{k1}(flag)))))) $$

Remember that E and E and also D and D cancel each other (if the same weak key is used) and also remember that

$k1 = k3$

so

$$ E_{k1}(D_{k2}(E_{k1}(E_{k1}(D_{k2}(E_{k1}(flag)))))) $$

We can now cancel the two middle encryption routines because they use the same weak key k1

$$ E_{k1}(D_{k2}(D_{k2}(E_{k1}(flag)))) $$

Now we can cancel the two middle decryption routines because they use the same weak key k2

$$ E_{k1}(E_{k1}(flag)) $$

And now we can cancel the last two encryption routines because they use the same key k1

$$ flag $$

from Crypto.Cipher import DES3
from Crypto.Util.Padding import pad
import os
import requests

SITE_ENC_FLAG = "https://aes.cryptohack.org/triple_des/encrypt_flag/{}"
SITE_ENC = "https://aes.cryptohack.org/triple_des/encrypt/{}/{}"

DES_WEAK_KEY_1 = "0000000000000000"
DES_WEAK_KEY_2 = "FEFEFEFEFEFEFEFE"

TRIPLE_DES_WEAK_KEY = DES_WEAK_KEY_1 + DES_WEAK_KEY_2 + DES_WEAK_KEY_1

enc_flag = requests.get(SITE_ENC_FLAG.format(TRIPLE_DES_WEAK_KEY)).json()["ciphertext"]

dec_flag = requests.get(SITE_ENC.format(TRIPLE_DES_WEAK_KEY, enc_flag)).json()["ciphertext"]

dec_flag = bytes.fromhex(dec_flag)

print(dec_flag)
