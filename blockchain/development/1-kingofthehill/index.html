1-KingOfTheHill

Why?

I learned about Ethereum and how it works and I wanted to try to interact with it. I decided to create a simple smart contract.

What?

The smart contract I decided to create is a “KingOfTheHill” donation smart contract. The contract gets donations and saves each address, the amount of the donations it sent, and a message of this address.

Addresses can interact with the smart contract by:

  • sending donations (payments)
  • getting the “King” address and its info (The “King” address is the one who donated the most amount of Ether)
  • getting the current address info (amount donated, message)
  • setting the message of the current address

The Owner of the contract can withdraw the donations at anytime

So first let’s create the struct AddressInformation, those are the fields saved for each address.

struct AddressInformation {
    uint256 donationAmount;
    string message;
}

Also now let’s create global variables which will be saved on the blockchain itself with the smart contract.

address king;

mapping(address => AddressInformation) public donationData;

I created a mapping variable which maps from address to AddressInformation struct. I also created an address type variable called king which will contain the current “King”.

The constructor is simple, just initializing the first “King”.

constructor(string memory _message){
    AddressInformation storage initializedInfo = donationData[msg.sender];
    
    initializedInfo.donationAmount = 0;
    initializedInfo.message = _message;

    king = msg.sender;
}

The Getter functions are very easy as well. The getKingOfTheHill just returns the AddressInformation and the address of the “king” variable.

The getMyDonationInfo just returns the current address (msg.sender) AddressInformation.

Keep in mind those functions are view type because they just get information from the contract and not setting anything.

function getKingOfTheHill() public view returns (address, uint256, string memory) {
    AddressInformation memory donationInfo = donationData[king];
    return (king, donationInfo.donationAmount, donationInfo.message);
}

function getMyDonationInfo() public view returns (uint256, string memory) {
    AddressInformation memory donationInfo = donationData[msg.sender];
    return (donationInfo.donationAmount, donationInfo.message);
}

We will create now a Setter function, this function just sets the message of the current address.

function setMyDonationMessage(string memory _message) public {
    AddressInformation storage donationInfo = donationData[msg.sender];

    require(donationInfo.donationAmount > 0, "Donation amount must be greater than zero");

    donationInfo.message = _message;
}

Now we need to create a function that will let people to donate coins there. This function is payable so it accepts coins being sends to it.

This function just get the current address donator, updates the donation amound, and if the current donation amount is more than the “King”, we got our new “King”!

function donate() public payable {
    require(msg.value > 0, "Donation amount must be greater than zero");

    AddressInformation storage donationInfo = donationData[msg.sender];
    AddressInformation memory kingDonationInfo = donationData[king];

    donationInfo.donationAmount += msg.value;

    if (donationInfo.donationAmount > kingDonationInfo.donationAmount)
    {
        king = msg.sender;
    }
}

We also want to add functions for just receiving coins without using the donate function

receive() external payable {
    donate();
}

fallback() external payable {
    donate();
}

At last we need to create the withdraw function, because I want only the owner of the contract to be able to withdraw coins from the donations, the contract I created inherit from “Owner.sol” contract.

So we need to update the constructor to call Owner() and to use this functionallity we need to add the modifier isOwner to the withdraw function

This function assures that the Owner is calling it, if it does it will transfer all the coins it has to the owner!

function withdraw() public isOwner {
    payable(owner).transfer(address(this).balance);
}

This was an example of very easy Ethereum contract, I will now show how to build tests and interact with it.

To add the UI for this contract I will just create a new react project, add the abi of the contract to the project as a json and import it inside App.js.

I will add a global variable called contractAddress which will contain the uploaded contract address.

Because I am not that good with frontend, I will create an ugly frontend that uses metamask.

All this application does is to show the blockchain information
