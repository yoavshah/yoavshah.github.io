Challenge

In this challenge we get a program that let’s us encrypt and decrypt values using AES-GCM. To get the flag we need to encrypt the string “give me the flag” and send it to the decrypt function with the appropriate $tag$, the problem is that in the encryption method, if the string “flag” is inside the plaintext the program will refuse signing our message!

Solution

First let’s talk about how AES-GCM works and why was it invented. As you can see in AES-CTR mode, modification to the ciphertext results in a direct modification to the plaintext (it’s a simple xor operation). If by any chance I know what the plaintext at specific index is, I can change the ciphertext to have full control over the plaintext.

Why use CTR instead of CBC? Because we can calculate each block independently and simultaneously - resulting in much faster encryption and decryption routines.

So what can we add to make the cipher resilient to modifications? Here comes AES-GCM which is a wrapper to AES-CTR.

When encrypting, AES-GCM calculates some HMAC function from the cipher called $tag$, when decrypting, the decryption function receives the $tag$ and verifies the ciphertext did not change.

Before we dive into AES-GCM we need to understand “Finite Fields”, please refer to the wikipedia page.

The most common examples of finite fields are given by n mod p where p is a prime number.

In our case, AES-GCM uses 128 bits per block, we want to transfer a block into an element in a field (I will explain later why). But how can we make a field of $2^{128}$ elements if it is obviously not a prime number (divided by 2)?

We could choose slightly bigger prime number but that will make it less efficient…

You can read about Non-prime fields here.

Given a prime power $q = p^n$, we can construct a field! In our case $p = 2$, $n = 128$ (because of our block size), and $q = 2^{128}$. In our field the elements will be polynomials of degree 128 or less.

First we need to choose an irreduciable polynomial of degree 128, in AES-GCM the chosen polynomial is $$ X^{128} + X^7 + X^2 + X + 1 $$ For example, in $GF(2^2)$, the irreduciable polynomial we can choose is $X^2 + X + 1$. A reduciable polynomial we cannot choose is $X^2 - 1$, because it can be reduced into $(X - 1)(X + 1)$ Another reduciable polynomial we cannot choose is $X^2 + 1$, because it can be reduced into $(X+1)^2 = X^2 + 2X + 1 = X^2 + 1$

In our field each number will be viewed as polynomial, let’s see an example under the field $GF(2^3)$ where the irreduciable polynomial is $P(x) = X^3 + X + 1$

$$ 5 = 0b101 = X^2 + 1 $$ $$ 4 = 0b100 = X^2 $$ Here is the addition method, it is actually the xor operation $\bigoplus$ $$ 5 + 4 = (X^2 + 1) + (X^2) = 2X^2 + 1 = 1 $$ Here is the multiplication method $\bigotimes$ $$ 5 * 4 = (X^2 + 1) * (X^2) = X^4 + X^2 $$ Mod it with $(X^3 + X + 1)$ and we get $X$ which is $2$ So $5 * 4 = 2$

To use finite fields in python I used galois and pyfinite libraries.

Now let’s have a deep look on the GCM mode. To gain understanding about GCM I watched this excellent video, viewed pycryptodome implementation and viewed another implementation (which returned the same values).

We first choose a key. We start with calculating $H$ (AuthKey), which will be used in signing the ciphertext. $$ H = ENC_{aes-ecb}(0, K) $$ Remember that H will be changed dramatically whenever $K$ is changed.

Now we will choose $nonce$, using this $nonce$ we will create AES-CTR with the same $K$ we choose before. Using $nonce$ we will calculate $E_0$ (First encryption block)

$$ E_0 = ENC_{aes-ctr}(K) $$

Now with those two variables, we will calculate the ciphertext and the signature of it, based on 4 parameters, $aad$ (associated data) and $plaintext$.

The encryption routine is the same as AES-CTR, it will generate a block and xor it with the plaintext.

We will sign the $ciphertext$ and the $aad$ together using $H$ and $E_0$, the whole signature algorithm uses $GF(2^{128})$ with the irreducable polynomial above. Each 16 bytes block translated to 128 bit block where the least significant bit is $X^{128}$.

$$ \begin{aligned} &tag = 0\\ &tag = tag + aad\\ &tag = tag * H\\ &tag = tag + C_1\\ &tag = tag * H\\ &……\ &tag = tag + C_n\\ &tag = tag * H\\ &tag = tag + L\\ &tag = tag * H\\ &tag = tag + E_0\\ \end{aligned} $$

After each operation we multiply $tag$ with $H$. First we add $aad$, then we iterate for each $ciphertext$ block and add it to the tag. After finished with all the $ciphertext$ blocks we create $L$ block which is the length of $aad$ and the length of the $ciphertext$ (both without their padding - their real length).

At the end we just add the $E_0$ block.

So we can actually create a mathematical equation for this calculation, let’s say we encrypt only 16 bytes of plaintext, so its ciphertext is just $C_1$, because it is a single block.

$$ tag = (((((aad * H) + C_1) * H) + L) * H) + E_0 $$ So $$ tag = aadH^3 + C_1H^2 + L*H + E_0 $$

Using this tag the decryption method can verify that there was no change to the ciphertext! Nice.

How can we hack this? In the given program the $IV$ or the $nonce$ does not change! Using this vulnerability we can find $H$ and $E_0$, thus creating our own verification $tag$.

Let’s use the encrypt method twice with 2 different plaintexts. The first one will be $M_a$ and the second one will be $M_b$. Both of them will have the size of 16 bytes, a single block.

Using the encrypt function we will get the $nonce$ (which is constant - this is the vulnerabillity), the $tag$, the $ciphertext$ and the $aad$.

Using the calculations above, we can understand that:

$$ tag_a = aadH^3 + C_aH^2 + LH + E_0 $$ $$ tag_b = aadH^3 + C_bH^2 + LH + E_0 $$

So $$ tag_a + tag_b = aadH^3 + C_aH^2 + LH + E_0 + aadH^3 + C_bH^2 + LH + E_0 $$ $$ tag_a + tag_b = 2aadH^3 + 2LH + 2E_0 + C_aH^2 + C_bH^2 $$ $$ tag_a + tag_b = C_aH^2 + C_bH^2 $$ $$ tag_a + tag_b = (C_a + C_b) * H^2 $$ $$ (tag_a + tag_b) * Inv(C_a + C_b) = H^2 $$ $$ H = SQRT((tag_a + tag_b) * Inv(C_a + C_b)) $$

So we get an equation of $H$ using 2 different $ciphertexts$ and their $tags$.

Now let’s calculate $E_0$ Using $H$ we can calculate $$ aadH^3 + C_aH^2 + LH $$ Because we have $tag_a$ and $$ tag_a = aadH^3 + C_aH^2 + LH + E_0 $$ We can calculate (The substract operator is $+$) $$ E_0 = tag_a + (aadH^3 + C_aH^2 + L*H) $$

Nice, now we got $E_0$ and $H$, using those two variables we can send the program the $plaintext$ "give me the flag", get the $ciphertext$ and then just calculate this $ciphertext$’s $tag$ alone

$$ tag_{flag} = aad*H^3 + C_{flag}H^2 + LH + E_0 $$

def encrypt(plaintext):
    
    return requests.get(SITE_ENC.format(plaintext.hex())).json()

def decrypt(nonce, ciphertext, tag, associated_data):

    get_params = SITE_DEC.format(nonce.hex(), ciphertext.hex(), tag.hex(), associated_data.hex())
    return requests.get(get_params).json()

def rev_bits(x):
    return int("{:0128b}".format(x)[::-1], 2)

irreducible_poly = (1 << 128) + (1 << 7) + (1 << 2) + (1 << 1) + (1 << 0)
GF = ffield.FField(128, gen=irreducible_poly)
F = galois.GF(2**128, irreducible_poly=irreducible_poly)

# Fields function
def FAdd(x, y):
    return rev_bits((F(rev_bits(x)) + F(rev_bits(y))).all())

def FMul(x, y):
    return rev_bits((F(rev_bits(x)) * F(rev_bits(y))).all())

def FInv(x):
    return rev_bits(GF.Inverse(rev_bits(x)))

def FPow(num, exp):
    total = 1 << 128
    while exp != 0:

        if exp % 2 == 0:
            num = FMul(num, num)
            
            exp = exp // 2
        
        else:
            total = FMul(total, num) 
            exp = exp - 1
            
    return total
