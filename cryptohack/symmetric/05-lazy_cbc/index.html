Challenge

In this challenge we get a program of a lazy developer whom used the KEY also as an IV, we can use the encrypt method which encrypts the data and the receive method which decrypt the data and returns it only if the data is not textual (we can force it to not be textual)

from Crypto.Cipher import AES


KEY = ?
FLAG = ?


@chal.route('/lazy_cbc/encrypt/<plaintext>/')
def encrypt(plaintext):
    plaintext = bytes.fromhex(plaintext)
    if len(plaintext) % 16 != 0:
        return {"error": "Data length must be multiple of 16"}

    cipher = AES.new(KEY, AES.MODE_CBC, KEY)
    encrypted = cipher.encrypt(plaintext)

    return {"ciphertext": encrypted.hex()}


@chal.route('/lazy_cbc/get_flag/<key>/')
def get_flag(key):
    key = bytes.fromhex(key)

    if key == KEY:
        return {"plaintext": FLAG.encode().hex()}
    else:
        return {"error": "invalid key"}


@chal.route('/lazy_cbc/receive/<ciphertext>/')
def receive(ciphertext):
    ciphertext = bytes.fromhex(ciphertext)
    if len(ciphertext) % 16 != 0:
        return {"error": "Data length must be multiple of 16"}

    cipher = AES.new(KEY, AES.MODE_CBC, KEY)
    decrypted = cipher.decrypt(ciphertext)

    try:
        decrypted.decode() # ensure plaintext is valid ascii
    except UnicodeDecodeError:
        return {"error": "Invalid plaintext: " + decrypted.hex()}

    return {"success": "Your message has been received"}

Solution

As we can see in the drawing above, in the encryption method the plaintext we send will be xored with the KEY and then will be processed in the AES block encryption routine. In addition, in the decryption method - the ciphertext will be processed in the AES block decryption routine and then will be xored with the KEY.

If we send to the encrypt method zero block after it is xored with the IV (which is the KEY), it will be inserted to the encryption routine, and we will get the ENC(KEY) as the cipher text.

[ZeroBlock]

Therefore, the first encrypted block will be equal to ENC([ZeroBlock] ⊕ IV) = ENC([ZeroBlock] ⊕ KEY) = ENC(KEY)

Now we have the the ENC(KEY), but how can we decrypt it? (we can’t just use the decrypt function because it will be xored with the IV - we will get the KEY ⊕ KEY which does not help us…)

We can just put it in the next block!

Let’s say we create a new ciphertext block that contains only zeros, the decryption of the first block will be non-sense, but CBC will use the cipher text first block (which is zeros) as the IV for the next block.

[ZeroBlock] [ENC(KEY)]

And now, the second decrypted block will be equal to DEC(ENC(KEY)) ⊕ [ZeroBlock] => KEY

from Crypto.Cipher import AES
import os
from Crypto.Util.Padding import pad, unpad
from datetime import datetime, timedelta
import requests

SITE_ENCRYPT = "https://aes.cryptohack.org/lazy_cbc/encrypt/{}"
SITE_DECRYPT = "https://aes.cryptohack.org/lazy_cbc/receive/{}"
SITE_GET_FLAG = "https://aes.cryptohack.org/lazy_cbc/get_flag/{}"

zero_block = "00" * 16 # 16 bytes

to_encrypt = zero_block
enc_key = requests.get(SITE_ENCRYPT.format(to_encrypt)).json()["ciphertext"]

to_decrypt = zero_block + enc_key
error_dec = requests.get(SITE_DECRYPT.format(to_decrypt)).json()["error"]

dec = error_dec[len("Invalid plaintext: "):]

first_block_decrypted = dec[:32]
second_block_decrypted = dec[32:64]

key = second_block_decrypted

flag = requests.get(SITE_GET_FLAG.format(key)).json()["plaintext"]

flag = bytes.fromhex(flag)

print(flag)
